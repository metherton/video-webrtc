Create a User Interface

Create a new directory somewhere on your development machine. Create a file named quickstart.html in it, with the following contents:

<html>
  <head>
    <title>Twilio Conversations - Video Quickstart</title>
    <link rel="stylesheet" href="//media.twiliocdn.com/sdk/quickstart/rtc-conversations-quickstart.min.css">
  </head>
  <body>
  <div id="remote-media"></div>
  <div id="controls">
    <div id="preview">
      <p class="instructions">Lookin' good!</p>
      <div id="local-media"></div>
      <button id="button-preview">Preview My Camera</button>
    </div>
    <div id="invite-controls">
      <p class="instructions">Invite an Endpoint</p>
      <input id="invite-to" type="text" placeholder="Address to send an invite to" />
      <button id="button-invite">Send Invite</button>
    </div>
    <div id="log">
      <p>&gt;&nbsp;<span id="log-content">Preparing to listen</span>...</p>
    </div>
  </div><!-- /controls -->
  <script src="//media.twiliocdn.com/sdk/rtc/js/v0.8/twilio-rtc-conversations.min.js"></script>
  <script src="quickstart.js"></script>
  </body>
</html>

This HTML is all we need to create and display a multi-party video Conversation.

Start a Web Server

For security reasons, some web browsers don't allow access to your camera and microphone when a page is loaded using the file:// protocol. Instead, we need to use the http:// or https:// protocol. That means we need to start a Web server.

Note: Firefox users, this step may not be required. First try opening the file from your file system.

There are many ways to start a web server and you may already have a preferred method. If that's the case, go for it! We'll meet you back here in a second. If not, you can try to launch a web server from your terminal using one of the commands below. Each command will serve up the contents of the current directory, so you need to execute it in the same location that you created quickstart.html. And be sure to kill the process when you're done!

Python 2.x:
$ python -m SimpleHTTPServer 8080

Python 3.x:
$ python -m http.server 8080

PHP 5.4+:
$ php -S 127.0.0.1:8080

Ruby 1.9.2+:
$ ruby -run -ehttpd . -p8080

Not working? Twilio has created scripts for Windows and Mac/Linux that will try these and other commands for you. You'll find the scripts in the quickstart directory of your beta download package. If you're having significant trouble serving up quickstart.html using a web server, you should consider installing Firefox and try simply opening the HTML file from the file system.
